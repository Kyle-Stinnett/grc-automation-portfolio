# Compliance Drift Detection

## Purpose
This project simulates continuous monitoring of security configurations to detect compliance drift.

## Scenario
In dynamic environments, configurations can change over time, leading to non-compliance with established controls.

## Approach
- Validate system settings against required controls
- Identify deviations (drift)
- Flag issues for remediation

## Example
Checks include:
- MFA enforcement
- Signed commit requirements

## GRC Value

This automation enables:

- Continuous monitoring of control effectiveness  
- Early detection of control failures  
- Reduced reliance on point-in-time audits  

## Business Impact

- Improves security posture through proactive detection  
- Reduces audit findings by identifying issues early  
- Supports scalable compliance in fast-moving environments
